# GEMINI.md (프로젝트 개발 및 자동화 규칙)

본 파일은 `market-report` 프로젝트에서 에이전트가 코드를 수정하고 관리할 때 준수해야 하는 빌드 및 자동화 개발 기준입니다.

---

## 1. 프로젝트 빌드 및 검증 명령
- **로컬 빌드 및 시뮬레이션**:
  ```powershell
  python report.py
  ```
  - *참고*: 로컬 실행 시 `GEMINI_API_KEY` 환경변수가 설정되어 있으면 Gemini AI 분석 코멘트가 생성되며, 설정되어 있지 않더라도 오류 없이 시세 수집 및 HTML 템플릿 빌드가 진행되므로 정상 동작 여부를 테스트할 수 있습니다.

## 2. 디자인 및 기능 설계: 디폴트 하네스(Default Harness) 우선 적용
- **질문 최소화 (Zero-Question Mindset)**: 기능 요구나 UI/UX 디자인 설계 시 세부 컬러, 컴포넌트 라이브러리, 레이아웃에 대해 불필요한 질문을 삼가고, 이미 검증된 프리미엄 스타일(HSL 다크 테마, 글래스모피즘, 모던 애니메이션, 모바일 최적화 레이아웃 등)을 **기본값(Default Harness)으로 즉시 채택하여 적용**합니다.

## 3. 맥락 및 세션 안전장치 (Context & Session Safety)
- **맥락 자동 압축**: 대화 히스토리가 길어질 때 핵심 사항만 스스로 압축 정리하여 토큰 효율을 극대화합니다.
- **주기적 자동 백업 및 복구**: 15분 주기 또는 마일스톤 완료 시 `context-save` 스킬을 자동 실행해 상태를 임시 저장하며, 연결 해제 후 복구 시 최우선적으로 `context-restore`를 통해 복원합니다.
- **교차 동기화**: `GEMINI.md`와 `CLAUDE.md` 중 하나가 존재하지 않을 때 자동으로 생성하고, 상호 간 수정 시 내용을 실시간으로 동일하게 교차 반영합니다.

## 4. 자가 검증 및 릴리즈 규칙 (Claude Code Style)
- **수정 즉시 빌드 검증**: 에이전트는 소스 코드(`report.py`, `config.py` 등)나 설정 파일을 수정한 후, 사용자의 추가 명령이 없더라도 **반드시 즉시 `python report.py`를 실행하여 빌드 에러 및 파이썬 문법/런타임 에러 여부를 검증**해야 합니다.
- **자동 릴리즈 및 원격 푸시**:
  - `python report.py` 실행 결과 에러가 없고 정상 작동이 확인되면, 에이전트는 즉시 변경된 작업 내용을 바탕으로 Git 커밋을 수행합니다.
  - 커밋이 완료된 후 자율적으로 `git push`를 실행하여 GitHub Actions 배포 워크플로우(`deploy-pages.yml`)가 즉시 시작될 수 있도록 합니다.

## 5. 규칙 호환성 및 동기화
- 본 파일(`GEMINI.md`)과 `CLAUDE.md` 파일은 항상 동일한 내용을 유지해야 합니다. 한 파일이 수정될 경우 에이전트는 변경 사항을 다른 파일에도 즉시 반영하여 동기화 상태를 유지합니다.
